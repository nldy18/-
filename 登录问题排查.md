# 登录问题排查指南

## 问题：开发模式下无法登录

### 第一步：检查服务器是否启动

**最重要：必须先启动服务器才能登录！**

1. 打开终端（Terminal）
2. 进入项目目录：
   ```bash
   cd /Users/bohu/Desktop/cousor/cs
   ```
3. 启动服务器：
   ```bash
   npm start
   ```
4. 看到以下信息表示启动成功：
   ```
   服务器运行在 http://localhost:3000
   注意：当前使用模拟微信登录，生产环境需要配置真实的微信开放平台信息
   ```

### 第二步：检查登录页面

1. 打开浏览器访问：`http://localhost:3000`
2. 会自动跳转到登录页面：`http://localhost:3000/login.html`
3. 查看页面上的"服务器状态"提示：
   - ✅ **绿色"服务器运行正常"** - 可以登录
   - ❌ **红色"无法连接到服务器"** - 需要先启动服务器

### 第三步：点击登录按钮

1. 点击"微信一键登录"按钮
2. 等待登录完成（通常1-2秒）
3. 登录成功后会自动跳转到日历页面

### 常见错误及解决方法

#### 错误1：无法连接到服务器

**现象：**
- 页面显示"无法连接到服务器"
- 点击登录按钮后显示网络错误

**解决方法：**
1. 确保已运行 `npm start` 启动服务器
2. 检查终端是否有错误信息
3. 确保端口3000没有被其他程序占用
4. 尝试访问 `http://localhost:3000/api/health` 检查服务器状态

#### 错误2：登录后页面空白

**现象：**
- 点击登录后跳转，但页面显示空白

**解决方法：**
1. 打开浏览器开发者工具（F12）
2. 查看Console标签页的错误信息
3. 检查Network标签页，看API请求是否成功
4. 清除浏览器缓存后重试

#### 错误3：数据库错误

**现象：**
- 服务器启动时报数据库错误

**解决方法：**
1. 检查项目目录是否有写入权限
2. 删除 `database.sqlite` 文件后重新启动（会重新创建）
3. 确保有足够的磁盘空间

### 验证服务器是否正常运行

在浏览器中访问以下URL，应该看到JSON响应：

- 健康检查：`http://localhost:3000/api/health`
  - 应该返回：`{"status":"ok","message":"服务器运行正常"}`

### 如果还是无法登录

1. **检查终端输出**：查看服务器启动时的错误信息
2. **检查浏览器控制台**：按F12打开开发者工具，查看错误信息
3. **检查网络请求**：在Network标签页查看API请求的响应
4. **重启服务器**：按Ctrl+C停止服务器，然后重新运行 `npm start`

### 快速测试命令

在终端运行以下命令测试服务器：

```bash
# 测试服务器是否响应
curl http://localhost:3000/api/health

# 应该返回：{"status":"ok","message":"服务器运行正常"}
```
