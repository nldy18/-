# 快速开始指南

## ⚠️ 重要提示

**必须先启动服务器才能登录！** 如果直接打开HTML文件是无法登录的。

## 第一步：安装依赖

```bash
cd /Users/bohu/Desktop/cousor/cs
npm install
```

## 第二步：启动服务器

```bash
npm start
```

看到以下信息表示启动成功：
```
服务器运行在 http://localhost:3000
注意：当前使用模拟微信登录，生产环境需要配置真实的微信开放平台信息
```

**⚠️ 保持终端窗口打开，不要关闭！**

## 第三步：访问应用并登录

1. **打开浏览器**访问：`http://localhost:3000`
   - ❌ 不要直接双击打开 `login.html` 文件
   - ✅ 必须通过 `http://localhost:3000` 访问

2. **会自动跳转到登录页面**

3. **查看服务器状态提示**：
   - ✅ 绿色"服务器运行正常" - 可以登录
   - ❌ 红色"无法连接到服务器" - 检查是否启动了服务器

4. **点击"微信一键登录"按钮**
   - 开发环境使用模拟登录，无需真实微信账号
   - 登录过程通常1-2秒

5. **登录成功后自动跳转到日历页面**

## 登录流程图示

```
终端运行 npm start
    ↓
浏览器访问 http://localhost:3000
    ↓
自动跳转到登录页面
    ↓
查看服务器状态（应该是绿色）
    ↓
点击"微信一键登录"
    ↓
登录成功，进入应用
```

## 常见问题

### Q: 点击登录没反应？
A: 检查服务器是否已启动（终端是否显示"服务器运行在..."）

### Q: 显示"无法连接到服务器"？
A: 
1. 确保已运行 `npm start`
2. 检查终端是否有错误信息
3. 确保访问的是 `http://localhost:3000` 而不是直接打开HTML文件

### Q: 登录后页面空白？
A: 
1. 按F12打开浏览器开发者工具
2. 查看Console标签页的错误信息
3. 清除浏览器缓存后重试

## 开发模式说明

- ✅ 开发环境默认使用**模拟微信登录**，无需配置微信开放平台
- ✅ 数据库文件会自动创建：`database.sqlite`
- ✅ 所有数据存储在数据库中，支持多用户

## 生产环境部署

1. 配置微信开放平台（获取AppID和AppSecret）
2. 设置环境变量（见README.md）
3. 使用HTTPS（微信登录要求）
4. 修改 `login.html` 中的 `wechatAppId`

## 常见问题

**Q: 登录后显示空白页面？**
A: 检查浏览器控制台，确保服务器正常运行

**Q: 无法保存任务？**
A: 检查网络连接和API接口是否正常

**Q: 数据库在哪里？**
A: 数据库文件 `database.sqlite` 在项目根目录
